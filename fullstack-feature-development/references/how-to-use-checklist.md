# 检查清单使用指南

## 模板的用途

这个检查清单是**AI和你的协作工具**，不是单纯的文档。

### 谁做什么？

| 角色 | 职责 |
|------|------|
| **AI (Kimi)** | 自动执行检查、运行验证脚本、发现问题 |
| **你 (开发者)** | 确认业务逻辑、处理异常、做决策 |

### 工作流程

```
你说"开发新功能" → AI自动开始
                          ↓
              AI: 复制模板到项目
                          ↓
              AI: 头脑风暴阶段（和你一起）
                          ↓
              AI: 后端开发 + 自动运行CP1/CP2/CP3
                          ↓
              AI: 前端开发 + 自动运行CP4
                          ↓
              AI: 集成测试 + 运行CP5
                          ↓
              AI: 生成完成报告，你确认
```

## 模板如何通用化

### 问题：原模板太具体
原模板写死了：
- `backend-node/src/index.js`
- `frontend-pc/src/store/`
- `getPool()` / `getAuthPool()`

这些是你的项目特有的，不能通用。

### 解决：分层设计

**第一层：通用检查点**（所有项目适用）
- CP1: 路由/接口是否注册
- CP2: API响应格式是否正确
- CP3: 数据库连接是否正确
- CP4: 前端数据流是否正确
- CP5: 端到端是否通

**第二层：项目适配**（AI自动检测）
- 后端入口文件在哪？→ AI自动找
- 数据库连接怎么叫？→ AI自动检测
- 前端目录结构？→ AI自动识别

**第三层：业务确认**（需要你确认）
- 功能边界对吗？
- 权限设计合理吗？
- 异常处理覆盖了吗？

## 实际使用场景

### 场景1: 你项目（Node + Vue）

你说："开发一个订单导出功能"

AI自动：
1. 检测到你的项目结构（backend-node / frontend-pc）
2. 按CP1-CP5执行，使用你项目特有的命令
3. 生成报告

### 场景2: 另一个项目（Python + React）

你说："开发一个数据统计功能"

AI自动：
1. 检测到不同结构（django-app / react-frontend）
2. 同样的CP1-CP5，但使用不同的验证命令
3. 生成报告

### 场景3: 简单功能（纯前端）

你说："加一个筛选组件"

AI自动：
1. 识别到无需后端
2. 跳过CP1/CP2/CP3
3. 只执行CP4（前端）和CP5（测试）

## 你什么时候需要动手？

|------|--------|----------|
| 代码检查 | ✅ 自动 | ❌ 不用 |
| 脚本运行 | ✅ 自动 | ❌ 不用 |
| 路由验证 | ✅ 自动 | ❌ 不用 |
| 边界确认 | ⚠️ 提示 | ✅ 确认/补充 |
| 业务逻辑 | ⚠️ 建议 | ✅ 决策 |
| 异常设计 | ⚠️ 提问 | ✅ 回答 |

## 关键改进

原设计的问题：
1. ❌ 给你一份空白模板让你填
2. ❌ 写死了项目结构
3. ❌ 不分AI和人，全部手动

新设计：
1. ✅ AI自动填写验证结果
2. ✅ 自动适配项目结构
3. ✅ 只有业务问题需要你确认

## 总结

**检查清单是给AI的"执行手册"，不是给你的"作业"**

你只需要：
1. 说"开发新功能"
2. 回答AI的业务问题（如果有）
3. 最终确认结果

AI负责：
1. 检测项目结构
2. 执行所有技术验证
3. 生成完整报告
