# 需求头脑风暴指南

## 目标
在写代码之前，彻底理解需求并识别所有检查点。

## 必问问题清单

### 1. 功能边界
- [ ] 这个功能解决什么业务问题？
- [ ] 用户是谁？不同角色权限一样吗？
- [ ] 有没有类似功能可以复用？

### 2. 数据流向
- [ ] 数据从哪来？（用户输入/数据库/API/文件）
- [ ] 数据到哪去？（展示/存储/导出/通知）
- [ ] 需要新建数据库表吗？
- [ ] 需要修改现有表结构吗？

### 3. 界面入口
- [ ] 用户从哪进入这个功能？（菜单/按钮/链接）
- [ ] 有列表页吗？有详情页吗？
- [ ] 有设置/配置页面吗？

### 4. API设计
- [ ] 需要几个API端点？
- [ ] 请求参数是什么？
- [ ] 响应数据格式？
- [ ] 需要分页吗？需要搜索吗？

### 5. 边界情况
- [ ] 没有数据时怎么显示？
- [ ] 数据量很大时怎么处理？
- [ ] 网络错误时怎么处理？
- [ ] 权限不足时怎么处理？

## 输出物

头脑风暴完成后，必须产出：

1. **设计文档** (`docs/design/[feature-name].md`)
   - 功能描述
   - 数据流图
   - API契约
   - 数据库变更

2. **检查清单** (复制 `assets/templates/dev-checklist.md`)
   - 已填写的串联检查点

3. **确认的问题列表**
   - 已澄清的边界问题
   - 待确认的问题（需要谁确认）
