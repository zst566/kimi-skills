# 用户使用指南 - 答疑解惑

## Q1: 检查清单是通用的吗？

**是通用的，但有自适应机制。**

通用的是**检查点逻辑**（CP0-CP5）：
- CP1: 接口是否注册
- CP2: API响应格式是否正确
- CP3: 数据库连接是否正确
- CP4: 前端数据流是否正确
- CP5: 端到端是否通畅

自适应的是**验证方式**：
- Node项目 → 检查 `app.use()`
- Python项目 → 检查 `urls.py`
- React项目 → 检查 `src/`
- Vue项目 → 检查 `src/store/`

AI会自动检测你的项目类型，选择对应的验证方式。

## Q2: 检查清单给谁用？我要填写吗？

**主要给AI用，你只需要确认关键决策。**

传统流程：
```
给你一份空白表格 → 你自己填 → 你自己检查 → 你报完成
```

这个技能流程：
```
你说"开发功能" → AI自动检测 → AI自动验证 → AI生成报告 → 你确认
```

**你只需要在3个地方动手**：
1. CP0设计阶段 → 确认AI理解的需求对吗？
2. 异常情况 → 检查点失败时决策
3. 最终交付 → 确认可以交付

其他全部由AI自动执行和填写。

## Q3: 如果我项目结构特殊怎么办？

**AI会检测，检测不到会问。**

检测顺序：
1. 找 `backend*/src/{app,index,server}.js`
2. 找 `frontend*/src/`
3. 找 `*/src/` 下的前端代码
4. 找 `server*/`

如果都找不到，AI会问你：
> "未检测到标准项目结构，请告诉我：
> - 后端入口文件在哪？
> - 前端代码目录在哪？"

然后AI会记住，后续使用你提供的路径。

## Q4: 所有功能都要走完整5个检查点吗？

**不一定，AI会自动判断。**

| 功能类型 | 执行的检查点 | 跳过的检查点 |
|---------|-------------|-------------|
| 完整功能（前后端+数据库） | CP0-CP5 | 无 |
| 纯前端组件 | CP0, CP4, CP5 | CP1, CP2, CP3 |
| 纯API接口 | CP0, CP1, CP2, CP3, CP5 | CP4 |
| 数据库迁移 | CP0, CP3 | CP1, CP2, CP4, CP5 |
| 配置修改 | CP0 | CP1-CP5 |

AI会根据你的需求自动判断哪些检查点是必须的。

## Q5: 实际使用示例

### 场景：你要开发一个新功能

**你说**：
```
我要开发一个订单导出功能
```

**AI自动执行**：
```
1. 检测项目结构
   → 发现 backend-node/src/app.js
   → 发现 frontend-pc/src/
   → 检测到 MySQL + getPool

2. 需求澄清（问你）
   → "导出哪些字段？"
   → "谁可以导出？"  
   → "导出Excel还是CSV？"

3. 生成检查清单 docs/checklist-订单导出.md

4. 后端开发 + 自动验证 CP1/CP2/CP3
   → 自动注册路由
   → 自动验证API响应
   → 自动检查数据库连接
   → 自动填充验证结果到清单

5. 前端开发 + 自动验证 CP4
   → 自动添加api.js方法
   → 自动检查response.code错误
   → 自动验证界面入口
   → 自动填充验证结果

6. 集成测试 + 自动验证 CP5
   → 运行端到端测试
   → 自动填充结果

7. 生成完成报告（问你确认）
   → "功能已通过全部检查点，确认交付？"
```

**你的工作量**：
- 回答3个业务问题
- 点一次确认

**AI的工作量**：
- 自动执行所有技术验证
- 填写完整检查清单
- 生成带证据的报告

## Q6: 如果检查点失败了怎么办？

**AI会停止，报告问题，请你决策。**

例如：
```
❌ CP3 检查失败：发现 getPool() 查询 users 表

问题：backend-node/src/services/userService.js:25
  const pool = getPool();
  await pool.execute('SELECT * FROM users...');

修复方案：
应该使用 getAuthPool() 而不是 getPool()

请选择：
1. 让AI自动修复
2. 我自己修复（修复后说"继续"）
3. 忽略此问题（不推荐）
```

## Q7: 可以用在其他项目吗？

**可以，这是全局技能。**

安装位置：`~/.config/agents/skills/`

任何项目都能触发：
- 你的其他项目
- 团队其他项目
- 开源项目
- 甚至临时任务

只要涉及开发，都会自动触发。
