# 技能使用场景指南

## 你能说什么来触发不同场景？

### 1. 功能验证场景（你的情况）

**功能开发完了，要做验收验证**

```
"验证一下SQLite缓存功能"
"检查查询历史功能是否完成"
"验收异步查询功能"
"帮我测试一下订单导出"
"确认功能可以上线了吗"
```

**AI会执行**：
- CP1: 检查路由是否注册
- CP2: 测试API响应格式
- CP3: 检查数据库连接
- CP4: 检查前端响应处理
- CP5: 引导你执行端到端测试

**产出**：验证报告（如上面的示例）

---

### 2. Code Review场景

**要审查已完成的代码**

```
"Code Review用户管理功能"
"审查查询缓存的代码"
"帮我检查一下这个PR"
"审查刚提交的代码"
```

**AI会执行**：
- CP1-CP4: 静态代码检查
- 代码质量检查（重复代码、错误处理等）
- 跳过CP5: 不执行运行测试

**产出**：代码审查报告

---

### 3. 上线前检查场景

**准备发布到生产环境**

```
"准备上线"
"发布前检查"
"生产环境准备"
"上线前最终检查"
```

**AI会执行**：
- CP1-CP5: 完整功能检查
- 部署特定检查（环境变量、数据库迁移等）
- 回滚方案检查

**产出**：上线检查清单

---

### 4. Bug修复场景

**修复已发现的问题**

```
"修复查询历史显示问题"
"修个bug：异步查询没反应"
"解决SQLite缓存的401错误"
```

**AI会执行**：
- CP0变种: 根因分析
- 修复代码
- CP5: 回归测试验证

**产出**：Bug修复报告

---

### 5. 新功能开发场景（原始场景）

**从零开发新功能**

```
"开发订单导出功能"
"实现用户管理模块"
"做一个数据统计功能"
```

**AI会执行**：
- CP0: 需求澄清
- CP1-CP4: 开发+自动验证
- CP5: 集成测试

**产出**：完整功能+检查清单

---

## 场景对比

| 场景 | 触发语句 | AI做什么 | 你做什么 | 产出 |
|------|---------|---------|---------|------|
| 功能验证 | "验证xxx功能" | 执行CP1-CP5 | 执行CP5的测试步骤 | 验证报告 |
| Code Review | "审查xxx代码" | 执行CP1-CP4+质量检查 | 确认审查意见 | 审查报告 |
| 上线前检查 | "准备上线" | CP1-CP5+部署检查 | 确认检查清单 | 上线清单 |
| Bug修复 | "修复xxx问题" | 根因分析+修复+CP5 | 描述问题+确认修复 | 修复报告 |
| 新功能开发 | "开发xxx功能" | CP0-CP5全流程 | 回答需求问题+确认交付 | 完整功能 |

## 关键区别

### 开发 vs 验证

| | 新功能开发 | 功能验证 |
|--|-----------|---------|
| **触发** | "开发xxx" | "验证xxx" |
| **CP0** | 需求澄清 | 跳过（已有设计） |
| **CP1-4** | AI写代码+验证 | 只验证不修改 |
| **CP5** | AI测试 | 引导你测试 |
| **产出** | 功能代码 | 验证报告 |

### 验证 vs Code Review

| | 功能验证 | Code Review |
|--|---------|-------------|
| **触发** | "验证xxx" | "审查xxx代码" |
| **CP5** | 执行（功能可用性） | 跳过（不运行） |
| **重点** | 功能是否可用 | 代码质量 |
| **产出** | 测试报告 | 审查意见 |

---

## 总结

这个技能**不只是开发工具**，而是：
- ✅ **开发助手**：从零开发新功能
- ✅ **验证工具**：验收已开发功能（你的情况）
- ✅ **审查助手**：Code Review
- ✅ **上线把关**：发布前检查
- ✅ **修复助手**：Bug修复

**你只需要说对关键词**，AI自动判断场景，执行对应的检查点组合。
